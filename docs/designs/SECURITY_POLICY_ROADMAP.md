# 安全策略深化路线图

更新时间：2026-05-05

类型：Design
归属层：system/auth
状态：Active

本文用于更新安全策略后续任务。MFA/TOTP 已进入已实现能力，不再作为“待补功能缺口”统计；后续重点转向登录告警、密码策略和风控。

---

## 1. 当前状态

已具备：

- 本地登录。
- access / refresh token。
- 会话管理。
- 安全中心。
- 最近登录日志。
- 密码修改。
- MFA/TOTP。
- 高敏操作二次验证。
- 登录失败锁定基础能力。
- 密码最小长度与数字/大写字母复杂度策略。
- 来源级失败节流与临时锁定。
- 安全事件落表与后台查询，当前覆盖来源锁定、账号锁定等真实认证风险。
- 历史密码复用限制，可通过 `security.password_history_limit` 配置启用。
- 密码过期提醒，可通过 `security.password_expire_days` 配置启用，并在安全中心暴露过期状态。

## 2. 后续 P1

### 2.1 登录告警

当前已补：

- 来源锁定、账号锁定写入 `system_auth_security_event`。
- 后台安全事件页可按账号、事件类型、风险等级查询。
- 未接入通知通道前，不展示“已发送邮件/短信”伪状态。

后续增强：

- 新设备登录提示。
- 异地或异常 IP 登录提示。
- 高频失败登录告警。

验收：

- 告警事件写入审计或安全事件。
- 前端安全中心能看到最近风险提示。
- 不在未实现通知通道前展示伪“已发送邮件/短信”。

### 2.2 密码策略

当前已补：

- 最小长度：已实现。
- 复杂度要求：已实现数字和大写字母两个可配置开关。
- 历史密码复用限制：已实现。
- 密码过期提醒：已实现。

验收：

- 策略从 `system/config` 读取，但校验语义属于 `system/auth`。
- 后端返回 error key。
- 前端只展示真实生效策略。

### 2.3 风控规则

当前已补：

- 按来源维度失败次数节流和临时锁定。
- 风控决策以安全事件形式可审计。

后续增强：

- 按用户、IP、设备指纹维度限流。
- 高风险登录要求 MFA。
- 高敏动作要求二次验证。

验收：

- 风控决策可审计。
- 用户提示不泄漏过多安全策略细节。
- 管理员可解释为什么被拦截。

## 3. SSO 边界

SSO / OAuth2 / OIDC 不属于当前已实现能力。

在身份源、回调域名、外部身份绑定策略未确定前：

- 登录页不得展示伪 SSO 按钮。
- 后端不得落半套 provider 表后不使用。
- 应先按 `docs/designs/SSO_OIDC_DESIGN.md` 完成设计评审。
